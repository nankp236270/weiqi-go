# 📁 docs 文件夹说明

本文件夹用于存放项目的辅助文档。

---

## 📂 文件夹结构

### bug-fixes/
存放所有bug修复相关的文档

- **`修复记录汇总.md`** - 所有bug修复的简要汇总（推荐阅读）
- **`archive/`** - 详细的修复文档归档（22-36号文档）

---

## 📖 如何使用

### 查看bug修复记录
1. 先阅读 `bug-fixes/修复记录汇总.md` 了解所有修复
2. 如需详细信息，查看 `bug-fixes/archive/` 中的具体文档

### 未来文档规范
- **重要文档** → 放在项目根目录（00-21编号）
- **Bug修复文档** → 放在 `docs/bug-fixes/archive/`
- **测试文档** → 放在 `docs/tests/`（如需要）
- **其他临时文档** → 放在 `docs/` 对应子文件夹

---

## 🎯 文档原则

1. **简洁** - 新文档保持简洁，重点突出
2. **分类** - 按类型放入对应文件夹
3. **汇总** - 同类文档创建汇总文件
4. **归档** - 详细文档放入 archive 子文件夹

---

**查看完整文档索引**: 请阅读根目录的 `00-文档索引.md`
